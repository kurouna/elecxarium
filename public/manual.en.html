elecxarium Manual

A browser-native, Terrarium-inspired ecosystem battle. You write a creature in TypeScript; it runs in a sandboxed Web Worker and competes against other players' creatures in one deterministic world.

1. The idea

Each tab is one creature — its species. You write its brain: a think() function the engine calls once per tick for every one of your creatures. It receives what that creature can see (its senses) and returns one action. Plants grow as free food, herbivores eat plants, carnivores hunt herbivores. The species that sustains the largest surviving population wins.

2. How to play

  • Tabs (top-left) — one tab = one player/species. + adds a species; the colored dot is that species' color (change it with the Color swatches).
  • Template ▾ — insert a starter creature into the active tab.
  • Run match — compiles every creature and starts a new match.
  • ▶ / ⏸ / ⏭ — play, pause, single-step. Speed sets ticks per second. Seed — each Run uses a fresh random seed. Tick lock to reuse the same seed every Run (so you replay the exact same match); type a number to choose a specific seed, or press 🎲 to roll a new random one.
  • 🎞 Replay — scrub the finished match on a timeline. ↺ resets.
  • 🏆 Tournament — runs the loaded creatures across 5 seeds and reports aggregate standings in the log.
  • Share — copies a URL that re-loads your exact creatures for someone else.
  • Leaderboard / Legend — live population, biomass, kills, births; the legend explains the arena's colors and shapes (each species, green = plant/food, grey = carcass). Hover a creature for its stats.

3. Roles & the food chain

A creature declares one role:

RoleEatsCan
herbivoreplantsmove, eat, reproduce, defend
carnivoreherbivores etc. — attack to kill, then eat the carcassmove, attack, eat, reproduce, defend
plant— (photosynthesises)reproduce only — rooted: can't move/attack/eat

A plant species is rooted and lives off photosynthesis (its energy income scales with the eatingSpeed trait); herbivores graze it like any other plant. The arena also auto-spawns simple background plants as baseline food, so a herbivore never depends on someone writing a plant. Carnivores can't eat a living animal directly — attack it until it dies, then eat the carcass.

4. Writing a creature

You export one defineCreature({...}). Imports resolve from @elecxarium/creature.

import { defineCreature, moveToward, nearest, eat } from '@elecxarium/creature';

interface Mem { target: { x: number; y: number } | null }

export default defineCreature<Mem>({
  meta: { name: 'MyGrazer', author: 'you', role: 'herbivore' },
  // 100 points to split across all 7 traits (the sum must be <= 100)
  traits: { maxEnergy: 25, eyesight: 25, speed: 20, attack: 0, defense: 10, eatingSpeed: 20, camouflage: 0 },
  // optional: custom SVG look (currentColor = your species color)
  appearance: { viewBox: '0 0 32 32', svg: '<circle cx="16" cy="16" r="11" fill="currentColor"/>' },
  initMemory: () => ({ target: null }),       // optional per-creature memory
  think(sense, mem) {                          // called once per creature per tick
    const plant = nearest(sense.nearby, (o) => o.kind === 'plant');
    if (!plant) return moveToward(sense.world.center);
    return plant.distance <= sense.self.reach ? eat(plant.id) : moveToward(plant.position);
  },
});

traits — the point budget

Seven integer traits, each ≥ 0, summing to ≤ 100 (over-budget creatures are rejected).

TraitEffect (at value v)
maxEnergyenergy capacity = 60 + v·1.4
eyesightsight radius = 40 + v·2.6 (how far nearby reaches)
speedmax move distance/tick = 4 + v·0.16 (moving costs energy)
attackdamage per hit = 4 + v·0.5 (carnivores only)
defenseflat damage reduction = v·0.25 (and defend() halves the hit)
eatingSpeedenergy gained per eat = 4 + v·0.26
camouflageshrinks the range at which others see you

think(sense, memory) → Action

Return exactly one Action each tick. sense is read-only; memory persists across ticks for that individual.

sense

FieldMeaning
sense.tickcurrent tick number
sense.self{ id, position, energy, energyMax, age, lifespan, reach, moveMax, sightRadius, canReproduce }
sense.world{ width, height, center }
sense.nearbyarray of visible things: { id, kind:'animal'|'plant'|'carcass', role?, species, isOwn, position, distance, energyState:'high'|'medium'|'low', isAlive }
sense.eventswhat happened to you since last tick: attacked, ateOk, eatFailed, born, reproduced
sense.random()deterministic random in [0, 1) (use this, not Math.random, for reproducible behavior)

5. Rules of the world

  • Energy drains every tick (more for larger, faster, moving creatures). Reaching 0 = death.
  • Reproduction needs energy ≥ 60% of capacity and a cooldown; it costs ~50% of max energy, so breed only when well-fed.
  • Death (starvation or old age, ~1500 ticks) leaves a carcass that carnivores can scavenge before it decays.
  • Reach for attack/eat is small (sense.self.reach) — you must be adjacent.
  • Match end: one species left, or 3000 ticks. Score = survival first, then time-integrated population (sustained dominance), then current biomass.

6. Tips

  • Don't reproduce the moment you can — eat first, breed when nearly full.
  • Herbivores: balance eyesight (find food), speed (reach it), eatingSpeed (refuel), and some defense/camouflage to survive predators.
  • Carnivores: attack + speed to catch prey, a little eatingSpeed to scavenge carcasses; keep maxEnergy low so you start hungry and hunt.
  • Use memory to remember a patrol target so you explore instead of standing still.
  • Use Tournament to test across seeds — a creature that wins one seed may lose another.

7. Safety & determinism

Your code runs in a Web Worker with no network, filesystem, DOM, or eval access — it only exchanges plain data (senses in, action out) with the engine. The whole simulation is deterministic given a seed, so replays and shared links reproduce the exact same match. Math.random is replaced by the seeded stream behind sense.random().
