elecxarium マニュアル

Microsoft .NET Terrarium 着想の、ブラウザ完結エコシステム対戦。生き物を TypeScript で書くと、 サンドボックス化された Web Worker 上で動き、1つの決定論的な世界で他プレイヤーのコードと競います。

1. コンセプト

各タブが1体の生き物（その種）です。あなたはその「脳」＝think() 関数を書きます。 エンジンは毎ティック、あなたの各個体に「見えているもの（senses）」を渡し、think() が「行動（action）」を 1つ返します。植物が無料の食料として生え、草食は植物を食べ、肉食は草食を狩ります。持続的に多くの個体を生存させた種が勝ちます。

2. 遊び方

  • タブ（左上） … 1タブ＝1プレイヤー＝1種。+で種を追加。色付きの点がその種の色 （Color スウォッチで変更可）。
  • Template ▾ … アクティブなタブにスターターを挿入。
  • Run match … 全種をコンパイルして新規対戦を開始。
  • ▶ / ⏸ / ⏭ … 再生・一時停止・1ティック送り。Speed は毎秒ティック数、 Seed … Run ごとに新しい乱数 seed を使用。lock すると毎回同じ seed を使い、同じ対戦を再現できます（値を直接入力して seed を指定／🎲 で新しい乱数）。
  • 🎞 リプレイ … 終了した対戦をタイムラインで再生。↺ でリセット。
  • 🏆 トーナメント … 読み込んだ種を5シードで総当たりし、集計順位をログに表示。
  • Share … その種一式を埋め込んだ URL をコピー（他の人がそのまま読み込めます）。
  • 順位 / 凡例 … 生存数・バイオマス・kills・births をライブ表示。凡例はアリーナの色と形の意味 （各種・緑＝植物/食料・灰＝死骸）を示します。個体にホバーすると詳細が出ます。

3. 役割と食物連鎖

生き物は role を1つ宣言します:

役割食べるものできること
herbivore（草食）植物移動・捕食・繁殖・防御
carnivore（肉食）草食動物など（attackで倒し、残った死骸をeat）移動・攻撃・捕食・繁殖・防御
plant（植物）—（光合成）繁殖のみ ― 根を張る（移動・攻撃・捕食は不可）

plant 種は根を張り、光合成で増えます（エネルギー収入は eatingSpeed に比例）。草食はこれを 他の植物と同様に食べます。アリーナには背景の植物も自動で生えるので、誰も植物を書かなくても草食が飢えることはありません。 肉食は生きた動物を直接は食べられず、attack で倒して死骸にしてから eat します。

4. 生き物を書く

defineCreature({...}) を1つ default export します。import は @elecxarium/creature から。

import { defineCreature, moveToward, nearest, eat } from '@elecxarium/creature';

interface Mem { target: { x: number; y: number } | null }

export default defineCreature<Mem>({
  meta: { name: 'MyGrazer', author: 'you', role: 'herbivore' },
  // 7つのトレイトに合計100ポイントを配分（合計 <= 100）
  traits: { maxEnergy: 25, eyesight: 25, speed: 20, attack: 0, defense: 10, eatingSpeed: 20, camouflage: 0 },
  // 任意: 見た目のSVG（currentColor が種の色になる）
  appearance: { viewBox: '0 0 32 32', svg: '<circle cx="16" cy="16" r="11" fill="currentColor"/>' },
  initMemory: () => ({ target: null }),       // 任意: 個体ごとの記憶
  think(sense, mem) {                          // 各個体・各ティックに1回呼ばれる
    const plant = nearest(sense.nearby, (o) => o.kind === 'plant');
    if (!plant) return moveToward(sense.world.center);
    return plant.distance <= sense.self.reach ? eat(plant.id) : moveToward(plant.position);
  },
});

traits — 持ち点

7つの整数トレイト（各 0 以上）、合計 100 以下（超過した種は読み込み拒否）。

トレイト効果（値 v のとき）
maxEnergyエネルギー上限 = 60 + v·1.4
eyesight視界半径 = 40 + v·2.6（nearby の届く距離）
speed1ティック最大移動 = 4 + v·0.16（移動はエネルギーを消費）
attack1撃のダメージ = 4 + v·0.5（肉食のみ）
defense被ダメージ軽減(一定) = v·0.25（さらに defend() でそのターンのダメージ半減）
eatingSpeed1回の捕食で得るエネルギー = 4 + v·0.26
camouflage相手から見つかる距離を縮める

think(sense, memory) → Action

毎ティック Action を1つ返します。sense は読み取り専用、memory は その個体でティックを跨いで保持されます。

sense

フィールド意味
sense.tick現在のティック番号
sense.self{ id, position, energy, energyMax, age, lifespan, reach, moveMax, sightRadius, canReproduce }
sense.world{ width, height, center }
sense.nearby視界内の配列: { id, kind:'animal'|'plant'|'carcass', role?, species, isOwn, position, distance, energyState:'high'|'medium'|'low', isAlive }
sense.events前ティック以降に自分に起きた事: attacked, ateOk, eatFailed, born, reproduced
sense.random()決定論的な乱数 [0,1)（再現性のため Math.random ではなくこれを使う）

アクションとヘルパ

Actionヘルパ効果
{ kind:'move', to }move(to) / moveToward(pos)地点へ向かう（moveMax で頭打ち）
{ kind:'attack', targetId }attack(id)肉食が隣接動物を噛む
{ kind:'eat', targetId }eat(id)隣接の植物（草食）or 死骸（肉食）を食べる
{ kind:'reproduce' }reproduce()子を産む（上限エネルギーの約50%を消費）
{ kind:'defend' }defend()このティックの被ダメージを半減
{ kind:'idle' }idle()何もしない

選択ヘルパ: nearest(list, pred?), farthest(list, pred?), dist(a, b), clampToWorld(pos, world)。

5. 世界のルール

  • エネルギーは毎ティック減少（大型・高速・移動が多いほど多く）。0で死亡。
  • 繁殖はエネルギーが上限の60%以上＋クールダウンが必要。上限の約50%を消費するので、満腹のときだけ繁殖を。
  • 死亡（餓死 or 寿命 ~1500ティック）は死骸を残し、肉食が腐敗前に漁れます。
  • 攻撃・捕食の到達距離は短い（sense.self.reach）。隣接が必要です。
  • 終了: 1種のみ生存 or 3000ティック。スコア＝生存 ＞ 時間積分個体数（持続的優勢）＞ 現バイオマス。

6. コツ

  • 繁殖できる瞬間に繁殖しない。まず食べて、ほぼ満タンで産む。
  • 草食: eyesight（餌発見）・speed（到達）・eatingSpeed（補給）に加え、 defense/camouflage で捕食者対策を。
  • 肉食: attack＋speed で獲物を捕え、少しの eatingSpeed で死骸を漁る。 maxEnergy を低めにして空腹で狩らせるのも手。
  • memory に巡回目標を覚えさせ、棒立ちにならず探索させる。
  • トーナメントで複数シードを検証。1シードで勝っても別シードで負けることがあります。

7. 安全性と決定論

あなたのコードはネットワーク・ファイル・DOM・eval 不可の Web Worker で動き、エンジンとは プレーンなデータ（senses 入力／action 出力）のみをやり取りします。シミュレーションは seed に対して完全に決定論的で、 リプレイや共有リンクは全く同じ対戦を再現します。Math.random は sense.random() の seeded ストリームに置き換えられます。
